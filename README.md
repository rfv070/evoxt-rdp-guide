## 为什么要用 RDP + VPS 这个组合？

先说说 RDP VPS 解决的是什么问题。

很多人需要一台永远在线、可以随时远程进入的 Windows 环境，场景包括但不限于：

- **挂机跑程序**：爬虫、自动化脚本、交易机器人，需要 7×24 小时不断线
- **海外 IP 访问**：业务需要从特定地区访问资源，本地网络搞不定
- **低配机器救星**：自己电脑性能不够，租一台高频 CPU 的 VPS 跑计算任务
- **多开软件**：需要同时跑多个账号或程序，一台机器不够用
- **团队共享工作台**：多人协作，登进同一台 Windows 环境干活

说白了就是：你需要一台不在你家、但完全属于你的 Windows 电脑。

## 怎么连接 Evoxt RDP？手把手教你

买完之后，Evoxt 会在大约 3 分钟内发邮件给你，里面包含三样东西：**IP 地址、用户名、密码**。

有了这三样，连接步骤如下：

### Windows 电脑连接 RDP（最简单）

1. 按 **Win + R**，输入 `mstsc`，回车
2. 在"计算机"栏填入你 VPS 的 IP 地址
3. 点击"连接"
4. 输入用户名（通常是 `Administrator`）和密码
5. 可能会弹出证书警告，点"是"继续
6. 进入 Windows 桌面，完成

整个过程不超过 1 分钟。

### Mac 连接 RDP

1. 去 App Store 下载 **Microsoft Remote Desktop**（微软官方免费应用）
2. 点击 "+"，选择"添加电脑"
3. 填入 IP 地址，保存
4. 双击连接，输入用户名密码
5. 进入桌面

### 手机端（iOS / Android）

App Store 或 Google Play 搜索"**Windows App**"（微软官方），下载后操作跟 Mac 类似。在外面偶尔需要看一眼服务器状态也很方便。

### 连不上？检查这几点

- 确认 Evoxt 控制台里服务器状态是"Running"
- 检查防火墙设置，确保 **3389 端口**开放（RDP 默认端口）
- Evoxt 控制台提供 **VNC 访问**，如果 RDP 实在连不上，可以先通过 VNC 进去看一眼
